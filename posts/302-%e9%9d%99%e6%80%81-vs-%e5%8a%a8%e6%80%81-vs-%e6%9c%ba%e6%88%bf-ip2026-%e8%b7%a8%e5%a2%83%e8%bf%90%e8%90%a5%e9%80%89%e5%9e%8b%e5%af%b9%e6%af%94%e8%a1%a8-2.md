## 三大 IP 类型分类

- **静态住宅 IP（Static Residential IP）**：固定归属家庭宽带用户的 IP，长期不变，ASN 是 ISP 住宅段；

- **动态住宅 IP（Rotating Residential IP）**：每次连接随机分配一个新的家庭 IP，IP 池超大；

- **机房 IP（Datacenter IP）**：服务器机房集中托管段，ASN 是 IDC 服务商。

## 适用场景决策树

### 第 1 问：账号是矩阵运营吗？

是 → 静态住宅 IP，每账号 1 个独占 IP，最稳；**否 → 进入第 2 问。

### 第 2 问：是长期任务还是短期任务？

长期（如每天 4 小时以上持续跑）→ 静态住宅 IP，避免每次断线重连 IP 切换；
短期（如 1 小时内完成）→ 动态住宅 IP，按流量付费划算；
内部工具 → 机房 IP 也能用。

### 第 3 问：目标平台对 IP 敏感吗？

是（LinkedIn、Facebook、TikTok、Amazon、Etsy）→ 必须静态住宅 IP；
中（YouTube、Reddit）→ 静态或动态都可，但静态更稳；
低（Twitter/X、Discord）→ 动态也凑合。

## 三种 IP 的真实场景对比

### 场景 1：亚马逊多店铺矩阵（3 店-30 店）

亚马逊 2026 年关联检测模型：登录 IP + 注册 IP + 支付 IP 三 IP 全部比对 ASN。如果同一 ASN 段登录过 3+ 账号，全部进 review。

- 静态住宅 IP**：3 店分别绑 3 个独立静态 IP，30 天实测 0 触发风控 ✅；

- **动态住宅 IP**：每次登录变 IP，亚马逊判”频繁异地登录”，14 天内 5 个店铺被警告 ✗；

- **机房 IP**：100% 触发 review，常见 30 天内全部 ban ✗。

### 场景 2：社媒矩阵 LinkedIn 12 号 90 天

LinkedIn 风控 2026 升级后，datacenter IP 一律 cancel 登录态；动态 IP 频繁切换触发设备指纹异常。

- **静态住宅 IP**：12 个号 90 天内掉线率 2/12（17%），根本原因硬件故障 ✗ ~85% 稳定；

- **动态住宅 IP**：12 个号 90 天内掉线率 9/12（75%），因为每次登录像换了台机器；

- **机房 IP**：100% 登不上。

### 场景 3：大模型 API 调用

OpenAI / Anthropic 2026 年对 datacenter ASN 调用自动降配（同 token 配额减半），对住宅 IP 维持正常。

- **静态住宅 IP**：长时间稳定调用 100% 配额；

- **动态住宅 IP**：单次请求有效，但每次 IP 切换 OpenAI 重新建模信任，配额按调用时长累计；

- **机房 IP**：配额降级 + 容易触发 429。
